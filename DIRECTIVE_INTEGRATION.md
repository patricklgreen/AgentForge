# Directive Framework Integration

AgentForge now integrates with the [deftai/directive](https://github.com/deftai/directive) framework to ensure uniform coding patterns and best practices across all generated projects.

## What Changed

### 🎯 **Consistent Standards**
Every project generated by AgentForge now follows the same high-quality coding standards:

- **Python**: ≥85% test coverage, ruff/black formatting, mypy strict typing, pytest testing
- **TypeScript**: Strict mode, ESLint, Prettier, Vitest testing, ≥85% coverage  
- **Go**: gofmt, golangci-lint, testify testing, ≥85% coverage
- **API Design**: RESTful conventions, OpenAPI documentation, proper status codes
- **Database**: Migrations, indexes, connection pooling, parameterized queries
- **Docker**: Multi-stage builds, non-root users, health checks, security hardening

### 📝 **Directive Notation**
All standards use clear priority notation:
- **!** = MUST (required, mandatory)
- **~** = SHOULD (recommended, strong preference)  
- **≉** = SHOULD NOT (discouraged, avoid unless justified)
- **⊗** = MUST NOT (forbidden, never do this)

### 🛠️ **Task Automation**
Projects now use **Taskfile.yml** instead of Makefiles:
```bash
task --list          # See available tasks
task check           # Run all quality checks
task test:coverage   # Verify ≥85% test coverage  
task dev             # Start development environment
task build           # Build the application
```

### 📋 **Generated Files**
Every project now includes:

1. **PROJECT.md** - Project-specific directive overrides and standards
2. **AGENTS.md** - Instructions for AI agents working on the project  
3. **Taskfile.yml** - Task automation with standard commands
4. **Comprehensive tests** - Unit, integration, with ≥85% coverage requirement
5. **Production-ready configs** - Docker, CI/CD, environment setup

## Architecture Changes

### 🧠 **Enhanced Agents**
All code-generating agents now receive directive context:
- **CodeGenerator**: Uses language-specific standards for all code
- **TestWriter**: Follows TDD principles with comprehensive coverage
- **DevOpsAgent**: Creates production-ready infrastructure  
- **Architect**: Designs following established patterns

### 🎛️ **Directive Service**
New `DirectiveService` provides:
- Language-specific coding standards
- Framework-specific best practices  
- API design patterns and security requirements
- Database and infrastructure standards
- Quality enforcement rules

### 📄 **Configuration Generation**
New `DirectiveConfigGenerator` creates:
- Project-specific directive files
- Task automation configurations
- Agent instruction files
- Quality gate definitions

## Implementation

The directive framework is automatically applied to all new projects. When agents generate code, they:

1. **Load directive context** based on project language/framework
2. **Apply consistent standards** using the directive notation
3. **Generate configuration files** for project management
4. **Ensure quality requirements** are met throughout

## Quality Gates

- **Type checking** (where applicable)
